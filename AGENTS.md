# ProfessionTips — Project Notes

Standalone addon extracted from the personal AllInOne addon (see ../AGENTS.md
for the general project rules: Classic Era API surface, Lua 5.1, no localized
name matching, user-facing text through `ns.L`). Planned for public release.

## What it does

Adds tooltip sections to profession reagents: per profession, the recipes an
item is used in, colored by difficulty at the character's current skill, with
the color-coded breakpoint scale ("1-44 64 84"), a "(not learned)" marker,
and processed-material chains (cloth -> bolt, ore -> bar, ...). All matching
is by item/spell ID; the only string comparison is localized-vs-localized
(profession spell name == skill line name), safe on every client language.

## Structure

```
ProfessionTips/
  ProfessionTips.toc   -- Vanilla (Classic Era 1.15.x, Interface 11507)
  Core.lua             -- bootstrap, ProfessionTipsDB (account-wide), ns.L,
                       -- ns.NewRecipeDB factory, ns.OnInit
  Options.lua          -- settings panel + /proftips; one checkbox per
                       -- profession that has data on this flavor
  ProfessionTips.lua   -- tooltip module (PROFESSIONS list, conversions,
                       -- rendering); professions without loaded data are
                       -- inert automatically
  Data/Vanilla/*.lua   -- one generated data file per profession
```

## Data pipeline (how the data files were built)

Generated by research agents from wago.tools DB2 CSV exports of the 1.15.x
wow_classic_era build (SkillLineAbility per SkillLine, SpellReagents,
SpellName, ItemSparse/ItemEffect/SpellEffect for recipe-item required
skill). Breakpoints: yellow = TrivialSkillLineRankLow, gray =
TrivialSkillLineRankHigh, green = floor((yellow+gray)/2), orange from
Wowhead Classic listview colors[0] (fallback: teaching item's
RequiredSkillRank). Excluded: profession rank spells, SoD-only entries
(spell IDs >= ~430000 and the 1.2M range), TBC+, never-obtainable oddities.
Every dataset was reconciled exactly against Wowhead's classic category
totals; regenerate the same way when the game data changes.

SkillLines: Alchemy 171, Blacksmithing 164, Cooking 185, Enchanting 333,
Engineering 202, First Aid 129, Leatherworking 165, Mining/Smelting 186,
Tailoring 197, Jewelcrafting 755 (TBC).

## TBC plan (not yet implemented)

Same code, per-flavor data:

1. Add `Data/TBC/*.lua` regenerated from the TBC classic build on
   wago.tools (recipes changed vs vanilla: new 300-375 recipes, adjusted
   breakpoints on some old ones) + `JewelcraftingData.lua`.
2. Add flavor-suffixed toc files: `ProfessionTips_Vanilla.toc` (Interface
   11508, loads Data/Vanilla) and `ProfessionTips_TBC.toc` (Interface
   20506 as of July 2026, loads Data/TBC). The clients pick the right toc
   automatically; the plain `ProfessionTips.toc` then serves as fallback.
3. Code is already flavor-agnostic: the PROFESSIONS list includes
   Jewelcrafting (skillSpell 25229), and both the tooltip module and the
   options panel skip professions whose data tables are not loaded.
4. Verify on the TBC client before release: modern Settings API and
   TooltipDataProcessor availability (the module already has a legacy
   tooltip fallback; Options.lua would need an InterfaceOptions fallback if
   the client turns out not to have the modern Settings API).
